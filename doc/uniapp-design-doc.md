## 一、需求分析

### 1.1 业务背景

东软智慧健康咨询系统是一款面向大众的健康咨询AI助手应用，提供智能健康问答、会员订阅、健康档案管理等功能。

### 1.2 核心功能需求

| 功能模块 | 功能描述 | 优先级 |
|---------|---------|--------|
| 用户登录 | 手机号验证码登录 | 高 |
| 健康咨询 | 与AI助手对话咨询 | 高 |
| 会员中心 | 会员状态、订阅方案、支付 | 高 |
| 健康档案 | 个人健康信息管理 | 中 |
| 咨询历史 | 查看历史对话记录 | 中 |
| 系统设置 | 用户设置、通知管理 | 低 |
| 支付结算 | 支付宝扫码支付、支付倒计时、支付成功 | 高 |
| AI报告解读 | 上传体检报告、AI指标分析、健康建议 | 高 |
| 药品查询 | 分类筛选、搜索过滤、药品详情 | 中 |
| 常见问题 | FAQ分类浏览、手风琴展开 | 低 |
| 消息通知 | 系统/会员/咨询通知、分类Tab、已读标记 | 中 |

### 1.3 用户角色

| 角色 | 描述 | 权限 |
|------|------|------|
| 普通用户（L0） | 未订阅会员 | 每日3次咨询 |
| 白银会员（L1） | 白银月/季/年卡 | 每日20次咨询 |
| 黄金会员（L2） | 黄金月/季/年卡 | 每日50次咨询 |
| 铂金会员（L3） | 铂金月/季/年卡 | 无限咨询 |

### 3.2 导航结构设计

#### 3.2.1 一级Tab导航

底部固定4个Tab：**首页** | **服务** | **会员** | **我的**

- 每个Tab页使用 `navigationStyle: custom` 自定义导航栏
- Tab切换通过 `uni.switchTab()` 实现
- 全局共享 `NavHeader` 和 `TabBar` 组件

#### 3.2.2 子页面导航

- 所有子页面左上角有**返回按钮**，点击调用 `uni.navigateBack()` 或 `navTo('来源页')`
- 已移除旧版"问诊"独立Tab——咨询页(`consult`)降为子页面
- 服务页(`services`)作为服务导航中心，列出所有核心服务入口

### 3.17 支付结算页 (payment/index)

**路由**：`/pages/payment/index`

**功能描述**：
- 展示订单信息（商品名称、编号、金额、有效期）
- 15分钟支付倒计时，超时自动取消

```typescript
export const useUserStore = defineStore('user', {
  state: () => ({
    user: null as User | null,
    token: '',
    isLoggedIn: false
  }),
  
  actions: {
    setUser(user: User) {
      this.user = user

      this.token = ''
      this.isLoggedIn = false
      uni.removeStorageSync('token')
    }
  },

    memberExpireTime: (state) => state.user?.memberExpireTime || ''
  }
})
```

```
进入咨询页 → 输入问题 → 点击发送 → 显示loading → 收到回复 → 显示消息 → 继续提问/结束
```

### 9.3 会员订阅流程

```
进入会员中心 → 选择方案 → 点击立即开通 → 创建订单 → 调用支付宝支付 → 支付成功 → 更新会员状态
```

### 9.4 退款流程

```
进入订单列表 → 选择订单 → 点击申请退款 → 填写原因 → 提交申请 → 等待审核 → 审核结果通知
```

### 9.5 紧急情况处理

```
用户发送消息 → 系统检测紧急关键词 → 弹出紧急提示 → 显示建议内容 → 引导拨打急救电话
```
