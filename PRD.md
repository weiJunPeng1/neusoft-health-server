## 二、产品背景与目标
### 2.1 业务背景
东软集团在医疗健康领域拥有30余年深厚积累，服务全国3000+家医疗机构，覆盖数亿患者。当前普通用户日常健康咨询需求日益增长，大量非紧急医疗咨询占用了医疗机构宝贵的资源，导致真正需要紧急救治的患者等待时间延长。

为解决这一痛点，东软集团决定开发智慧健康咨询系统，通过接入DeepSeek大模型，为用户提供7×24小时不间断的健康科普、症状咨询、用药指导和急救常识服务，有效分流非紧急医疗需求，提升医疗资源利用效率。

### 2.2 产品定位
面向普通大众的**轻量级健康咨询与科普平台**，作为医疗机构的补充和延伸，为用户提供便捷、专业、安全的日常健康咨询服务。

### 2.3 核心价值
- **对用户**：随时随地获取专业健康建议，节省就医时间和成本
- **对医疗机构**：分流非紧急咨询，集中资源处理重症患者
- **对东软**：拓展医疗健康C端服务，提升品牌影响力

### 2.4 可量化成功指标
- **用户指标**：上线3个月累计注册用户≥10万人，日活跃用户≥5000人，用户留存率≥30%
- **业务指标**：日均咨询量≥2000次，人均咨询次数≥3次
- **体验指标**：AI咨询响应时间≤3秒(95%分位)，用户满意度≥4.2分(5分制)
- **技术指标**：系统可用性≥99.9%，故障恢复时间≤30分钟
- **合规指标**：100%符合《个人信息保护法》《数据安全法》《互联网医疗健康信息服务管理办法》要求

### 2.5 上线时间要求
- 需求评审：2026-05-20
- 开发周期：2026-05-21 ~ 2026-06-16(4周)
- 测试周期：2026-06-17 ~ 2026-06-23(1周)
- 正式上线：2026-06-24

## 四、核心业务流程图
### 4.1 用户AI咨询主流程
```
用户进入系统 → 首次进入强制弹窗确认免责声明 → 进入首页
    ↓
点击AI助手入口/点击常见问题 → 进入咨询页面
    ↓
用户输入问题/点击快捷问题 → 输入内容敏感词过滤
    ↓
┌─ 过滤不通过 → 提示违规内容并拦截 ─┐
│                                    ↓
过滤通过 → 显示用户消息气泡 → 显示"AI正在思考..."加载动画
    ↓
调用DeepSeek API(携带系统提示词+对话上下文)
    ↓
┌─ API调用失败 → 自动重试3次 → 仍失败返回友好提示 ─┐
│                                                    ↓
API调用成功 → 输出内容安全审核
    ↓
┌─ 审核不通过 → 返回标准化安全提示 ─┐
│                                    ↓
审核通过 → 添加强制医疗风险提示 → 显示AI回复气泡
    ↓
保存对话记录 → 更新会话信息 → 结束
```

### 4.2 紧急情况处理流程
```
用户输入问题 → 系统匹配紧急关键词库(心梗/中风/大出血/猝死等)
    ↓
匹配成功 → 立即终止正常咨询流程
    ↓
弹出全屏红色紧急提示弹窗
    ↓
显示"⚠️ 您描述的情况可能危及生命，请立即拨打120急救电话！"
    ↓
提供对应紧急情况的基础急救常识(如心肺复苏步骤)
    ↓
禁止AI生成任何其他回复 → 记录紧急情况日志 → 结束
```

### 4.3 用户注册登录流程
```
用户进入系统 → 检测到未登录 → 跳转到登录页面
    ↓
用户输入手机号 → 点击"获取验证码"
    ↓
系统验证手机号格式 → 发送6位数字验证码(有效期5分钟)
    ↓
用户输入验证码 → 点击"登录"
    ↓
系统验证验证码 → 新用户自动创建账号
    ↓
生成JWT令牌(有效期7天) → 存储到本地
    ↓
跳转到首页 → 结束
```

## 五、详细功能需求
### 5.1 系统基础功能模块(P0 核心必选)
#### 5.1.1 用户登录注册
- **功能描述**：支持用户通过手机号验证码和第三方快捷方式登录注册
- **前置条件**：用户首次进入系统或登录状态过期
- **操作流程**：
  1. 用户输入11位中国大陆手机号
  2. 点击"获取验证码"，系统验证手机号格式并发送验证码
  3. 用户输入6位数字验证码，点击"登录"
  4. 系统验证验证码，新用户自动完成注册
  5. 生成JWT令牌，返回给客户端存储
- **异常处理**：
  - 手机号格式错误：提示"请输入正确的11位手机号"
  - 验证码发送频繁：提示"验证码发送过于频繁，请1分钟后再试"
  - 验证码错误：提示"验证码错误，请重新输入"
  - 验证码过期：提示"验证码已过期，请重新获取"
- **验收标准**：
  - 支持中国大陆11位手机号登录注册
  - 验证码有效期5分钟，1分钟内最多发送3次
  - 登录成功后自动跳转到首页
  - 7天内无需重新登录
  - 支持微信、支付宝第三方快捷登录

#### 5.1.2 系统首页
- **页面元素**：
  1. 顶部：系统logo、搜索框
  2. 中部：AI助手入口卡片(大按钮)、常见问题分类快捷入口(6个)
  3. 底部：健康资讯轮播(3条)、固定免责声明提示条
- **交互逻辑**：
  - 点击AI助手入口：进入咨询页面
  - 点击常见问题分类：进入咨询页面并自动发送对应问题
  - 点击健康资讯：跳转到资讯详情页(H5)
- **验收标准**：
  - 页面加载时间≤2秒
  - 所有元素在320px~1920px屏幕尺寸下显示正常
  - 点击所有入口跳转正确
  - 免责声明固定在页面底部，不随滚动消失

#### 5.1.3 全局免责声明
- **功能描述**：在系统所有关键位置展示免责声明，明确系统能力边界
- **展示位置**：
  1. 首次进入系统强制弹窗确认
  2. 所有页面底部固定显示
  3. 咨询页面顶部固定显示
  4. 所有AI回复开头自动添加
- **内容**："【健康咨询仅供参考，不能替代专业医生诊断和治疗】"
- **验收标准**：
  - 首次进入系统必须点击"我已阅读并同意"才能继续使用
  - 所有AI回复都包含规定的免责声明
  - 免责声明文字清晰可见，不可隐藏

### 5.2 AI健康咨询核心功能模块(P0 核心必选)
#### 5.2.1 文本对话咨询
- **功能描述**：支持用户输入文字进行健康咨询，DeepSeek模型实时回复
- **前置条件**：用户已登录并确认免责声明
- **操作流程**：
  1. 用户在输入框输入文字(最多500字)
  2. 点击"发送"按钮或按回车键
  3. 系统对输入内容进行敏感词过滤
  4. 过滤通过后，显示用户消息气泡
  5. 显示"AI正在思考..."加载动画
  6. 调用DeepSeek API，传入系统提示词和最近10轮对话上下文
  7. 接收API响应，对输出内容进行安全审核
  8. 审核通过后，添加免责声明前缀
  9. 显示AI回复消息气泡，保存对话记录
- **系统提示词(固定不可修改)**：
  ```
  你是东软智慧健康咨询助手，仅能提供健康科普和咨询服务。
  严禁提供任何疾病诊断、治疗方案、处方建议或手术建议。
  所有回复开头必须添加："【健康咨询仅供参考，不能替代专业医生诊断和治疗】"
  如用户描述紧急医疗情况(心梗、中风、大出血、呼吸困难等)，请立即回复："⚠️ 您描述的情况可能危及生命，请立即拨打120急救电话！"并提供基础急救常识。
  如用户询问超出健康咨询范围的问题，请礼貌拒绝回答。
  回答要简洁明了，通俗易懂，避免使用专业术语。
  ```
- **异常处理**：
  - 输入内容违规：提示"您的输入包含违规内容，请修改后重新发送"
  - API调用失败：自动重试3次，仍失败则提示"系统繁忙，请稍后再试"
  - 输出内容违规：返回"抱歉，我无法为您提供相关建议，请咨询专业医生"
- **验收标准**：
  - 单条咨询响应时间≤3秒(95%分位)
  - 所有AI回复都包含规定的免责声明
  - 紧急情况识别准确率≥99%
  - 违规内容拦截率≥99%
  - 支持多轮对话上下文理解

#### 5.2.2 对话历史记录
- **功能描述**：自动保存用户所有咨询对话，支持查看和管理
- **操作流程**：
  1. 用户进入个人中心，点击"咨询历史"
  2. 按时间倒序展示所有咨询会话列表
  3. 每个会话显示标题(截取第一条问题前20字)、最后咨询时间、最后一条消息预览
  4. 点击会话进入详情页，查看完整对话历史
  5. 支持左滑删除单条会话
  6. 支持一键清空所有历史记录
- **验收标准**：
  - 会话列表加载时间≤1秒
  - 历史对话显示完整，顺序正确
  - 删除操作成功后，数据不可恢复
  - 支持分页加载更多历史记录

#### 5.2.3 常见问题快捷提问
- **功能描述**：预设高频健康问题，用户点击即可快速提问
- **问题分类**：
  1. 感冒发烧
  2. 肠胃不适
  3. 睡眠问题
  4. 运动健康
  5. 饮食营养
  6. 皮肤问题
- **验收标准**：
  - 每个分类下至少有5个常见问题
  - 点击问题自动进入咨询页面并发送
  - 支持后台动态添加、编辑、删除常见问题

#### 5.2.4 内容安全过滤
- **功能描述**：对用户输入和AI输出进行双重安全过滤，防范违规内容
- **过滤规则**：
  1. 敏感词过滤：政治、色情、暴力、医疗风险等敏感词
  2. 医疗风险过滤：禁止AI提供诊断、治疗、处方建议
  3. 违法违规内容过滤：禁止讨论违法犯罪活动
- **验收标准**：
  - 敏感词拦截率≥99%
  - 医疗风险内容拦截率≥99%
  - 违规内容记录到日志，便于审计

### 5.3 用户管理模块(P0 核心必选)
#### 5.3.1 个人信息管理
- **功能描述**：用户可以查看和修改个人基本信息
- **可修改信息**：
  - 头像：支持上传本地图片，自动压缩裁剪为200×200px
  - 昵称：2-20个字符，支持中文、英文、数字和下划线
  - 性别：男/女/未知
  - 年龄：0-120岁
- **不可修改信息**：
  - 手机号：如需修改需联系客服
  - 用户ID：系统自动生成，唯一标识
- **验收标准**：
  - 个人信息展示正确
  - 修改信息后实时生效
  - 昵称包含敏感词时提示并拒绝修改

#### 5.3.2 健康档案管理
- **功能描述**：用户可以填写基础健康信息，用于AI提供更精准的建议
- **健康信息项**：
  - 身高(cm)
  - 体重(kg)
  - 血型(A/B/AB/O/未知)
  - 过敏史
  - 既往病史
- **隐私设置**：
  - 支持设置是否自动同步健康档案给AI
  - 支持随时清除健康档案数据
- **验收标准**：
  - 健康信息加密存储
  - 自动同步开启后，AI咨询时会自动携带健康信息
  - 清除健康档案后，数据不可恢复

#### 5.3.3 个人数据管理
- **功能描述**：用户可以管理自己的个人数据，包括咨询历史、收藏内容等
- **功能点**：
  - 查看咨询历史记录
  - 删除单条/全部咨询历史
  - 收藏有用的AI回复
  - 查看收藏内容
  - 取消收藏
  - 导出个人数据(支持JSON和PDF格式)
- **验收标准**：
  - 收藏功能正常
  - 导出的数据包含所有个人信息
  - 删除数据后，系统不再保留相关记录

#### 5.3.4 系统设置
- **功能描述**：用户可以根据自己的喜好设置系统参数
- **设置项**：
  - 消息通知设置：开启/关闭系统通知、咨询回复提醒
  - 语音设置：语音播报语速、音量、音色
  - 隐私设置：开启/关闭匿名咨询、是否允许推荐健康科普
  - 缓存管理：清除应用本地缓存
  - 账号安全：修改密码、退出登录、退出所有设备
- **验收标准**：
  - 所有设置项修改后实时生效
  - 退出登录后清除本地令牌
  - 清除缓存后不影响用户数据

### 5.5 安全与合规功能模块(P0 强制要求)
#### 5.5.1 数据加密
- **功能描述**：对用户敏感数据进行加密存储和传输
- **加密策略**：
  - 手机号、健康档案、咨询记录等敏感数据采用AES-256加密存储
  - 密码使用BCrypt不可逆加密算法
  - 所有数据传输采用HTTPS/TLS 1.3加密
  - 数据库备份文件加密存储
- **验收标准**：
  - 数据库中存储的敏感数据为密文
  - 网络传输的数据为加密状态
  - 加密密钥存储在环境变量中，不硬编码在代码里

#### 5.5.2 数据脱敏
- **功能描述**：对敏感信息进行脱敏展示，防止信息泄露
- **脱敏规则**：
  - 手机号：138****1234
  - 身份证号：410101********1234
  - 银行卡号：6222 **** **** 1234
- **脱敏范围**：
  - 所有页面展示的敏感信息
  - 所有日志中输出的敏感信息
  - 导出的用户数据(默认脱敏)
- **验收标准**：
  - 所有敏感信息都按规则脱敏展示
  - 只有超级管理员才能查看明文敏感信息

#### 5.5.3 权限控制
- **功能描述**：基于RBAC模型实现细粒度权限控制
- **权限粒度**：
  - 菜单权限：控制不同角色可以访问的菜单
  - 按钮权限：控制不同角色可以操作的按钮
  - 数据权限：控制不同角色可以查看的数据范围
- **验收标准**：
  - 不同角色只能访问其权限范围内的功能
  - 越权访问被拦截并提示
  - 权限修改后实时生效

#### 5.5.4 接口安全
- **功能描述**：保护API接口安全，防止恶意调用
- **安全措施**：
  - 所有API接口进行身份认证
  - 接口限流：单个用户每分钟最多发送60条咨询消息
  - 防止SQL注入：使用预编译语句
  - 防止XSS攻击：对输入输出内容进行转义
  - 防止CSRF攻击：使用CSRF Token
- **验收标准**：
  - 未认证用户无法访问API接口
  - 超过限流限制的请求被拦截
  - 常见的Web安全漏洞扫描无高危漏洞

#### 5.5.5 数据备份与恢复
- **功能描述**：定期备份系统数据，确保数据安全
- **备份策略**：
  - MySQL每日凌晨2点进行全量备份
  - 每小时进行增量备份
  - 备份文件保留30天
  - 备份文件存储在异地服务器
- **验收标准**：
  - 备份任务自动执行
  - 备份文件完整可用
  - 数据恢复时间≤30分钟

### 8.4 UI验收标准
- 严格遵循设计规范，色彩、字体、间距统一
- 所有交互元素有明确的反馈效果
- 加载状态和空状态有友好的提示
- 表单验证实时进行，错误提示清晰明确
- 符合WCAG 2.1 AA可访问性标准
